# Создание простейшего проекта

В этом уроке мы настроим среду разработки и создадим простую консольную игру.

## Подготовка

1. Зарегистрируйся на сайте [DeepSeek](https://chat.deepseek.com/). Это бесплатный искусственный интеллект, к которому ты сможешь обращаться по любым вопросам. Если у тебя будут возникать сложности с любой частью задания — попробуй спросить DeepSeek. Если он не сможет дать ответ, тогда заходи ко мне. Но помни, что ИИ может ошибаться — не доверяй ему слепо.
2. Установи [PyCharm Community](https://www.jetbrains.com/pycharm/download/). Это интегрированная среда разработки (IDE) для программирования на Python. Все задания ты можешь делать в ней.
3. Зарегистрируйся на сайте [GitHub](https://github.com/). Это бесплатный репозиторий для хранения кода. После регистрации склонируй репозиторий, в котором ты сейчас находишься, к себе на компьютер через интерфейс PyCharm. DeepSeek подскажет, как это сделать.

## Написание кода

1. Создай папку `src` — в ней мы будем хранить исходные файлы нашей будущей программы.
2. В папке `src` создай файл `main.py` — пока это будет единственный файл нашей будущей программы.
3. Напиши в этом файле консольную игру «Виселица» (по каждому шагу можешь консультироваться с DeepSeek, но постарайся понять суть того кода, который ты будешь писать):
   1. Объяви массив со списком слов, которые программа может загадать игроку. Задай туда любые слова.
   2. Реализуй выбор случайного слова из этого массива.
   3. Реализуй игровой цикл. Этот цикл будет выполняться до тех пор, пока пользователь не угадает слово или пока он не совершит 10 ошибок. В цикле:
      1. Выведи на экран загаданное слово, закрыв все буквы, которые игрок ещё не угадал, звёздочками (на первой итерации все буквы будут закрыты).
      2. Попроси игрока ввести букву.
      3. Проверь, есть ли введённая буква в выбранном ранее случайном слове. Если буквы нет, увеличь счётчик ошибок. Если буква есть, она должна открыться в загаданном слове на следующей итерации.

## Тестирование программы

1. Создай папку `tests` — в ней мы будем хранить тесты нашей программы.
2. В папке `tests` создай папку `manual` — в этой папке мы будем хранить инструкции для ручного тестирования.
3. В папке `manual` создай файл `main.md` — это будет описание основного процесса ручного тестирования нашей программы.
4. Опиши в созданном файле все проверки, которые, как тебе кажется, нужно сделать, чтобы полноценно проверить программу.
5. Выполни описанные проверки. Если найдёшь ошибки, исправь их и повтори тестирование заново.

## Создание пулл-реквеста

Когда программа будет написана и протестирована, выполни следующие действия, чтобы отправить результаты мне:

1. В интерфейсе PyCharm создай ветку `1-first-project`.
2. В этой ветке создай коммит с сообщением «Add first project».
3. Сделай push в эту ветку.
4. В интерфейсе GitHub открой pull request.
